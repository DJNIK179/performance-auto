# Демонстрация сайта по туннелю

## Вариант 1: Одна команда (сервер + туннель)

```bash
npm run demo
```

Команда сама проверит порт `3000`: если локальный сервер уже работает, она использует его; если нет, поднимет новый. Затем запустится туннель, и в консоли появится ссылка **https://xxxxx.tunnelmole.net** — отправьте её тому, кому показываете сайт.

---

## Вариант 2: По шагам

### 1. Локальный сервер (для себя или для туннеля)

```bash
npm run dev
```

Откроется браузер на http://localhost:3000. Сервер работает на порту 3000.

### 2. Туннель (второй терминал)

Сначала запустите `npm run dev` в первом терминале, затем:

```bash
npm run tunnel
```

Скопируйте ссылку **https://xxxxx.tunnelmole.net** и отправьте её.

### Альтернативы

Если основной вариант не подходит, можно попробовать:

```bash
npm run tunnel:localtunnel
```

или:

```bash
npm run tunnel:cloudflare
```

В этом проекте основной вариант теперь `tunnelmole`, потому что он не просит tunnel password и у меня отработал стабильнее, чем `localtunnel` и `Cloudflare quick tunnel`.

---

*Остановить: `Ctrl+C` в терминале. Если работаешь в Git Bash на Windows и процесс не завершается, используй `npm run stop`. Иногда Git Bash показывает скрытое подтверждение остановки batch-процесса и ждёт `Y` + Enter.*

Если изображения через публичную ссылку начинают "висеть", чаще всего это значит, что сам туннель завершился. В новой версии `npm run demo` сервер и туннель запускаются без `concurrently`, чтобы это происходило реже.

**502 Bad Gateway** — туннель запущен, а сервер не работает. Запустите `npm run dev` (или `npm start`) и обновите страницу по ссылке туннеля.

**503 Service Unavailable** — обычно это проблема самого туннельного сервиса, а не сайта. Проще всего перезапустить `npm run demo` и получить новую ссылку.

**530 / origin unregistered** — это типичная ошибка Cloudflare Tunnel, когда туннель отключился или сеть режет соединение. В таком случае лучше использовать обычный `npm run demo`.
